3. Adequação ao padrão do projeto
-----------------------------------------
Foi identificado que os controllers existentes utilizam
funções exportadas diretamente, e não classes.

Exemplo do padrão utilizado:

export async function Inscrever(req, res) {}

Por esse motivo o TaskController foi adaptado para seguir
o mesmo padrão arquitetural já utilizado no projeto.


4. Correção das rotas existentes
-----------------------------------------
Foi identificado que algumas rotas chamavam os controllers
sem enviar os parâmetros req e res.

Exemplo incorreto:

return Inscrever()

Correção aplicada:

return Inscrever(req, res)

Ajuste realizado nas rotas:
- /token-enviado
- /inscrever
- /verificar-logiin
- /mudar-senha
- /desativar-conta
- /reativar-conta


5. Integração com autenticação
-----------------------------------------
Foi analisado o middleware de autenticação.

Verificado que após validar o JWT o sistema injeta:

req.userId

Portanto o usuário criador da tarefa passa a ser obtido
automaticamente pelo token autenticado.

Exemplo:

criadoPor: req.userId

Isso evita que o front-end informe manualmente o ID
do usuário, aumentando a segurança da aplicação.


6. Criação da rota de tarefa

- Active

Fluxo:
